autoSize {MSImaging} R Documentation

Calculate recommended canvas size for MSImaging visualization

Description

Usage

autoSize(dims, padding,
    scale = 1,
    is.multiple.combine.wide = FALSE,
    ratio.threshold = 1.25,
    ratio.scale = 1.5);

Arguments

dims

The dimensions of MSImaging raw data. Can be either: 1. A numeric vector with at least two elements representing [width, height] 2. A list object containing at least two named elements "w" and "h".

padding

A numeric vector specifying CSS-style padding in the order: [top, right, bottom, left]. Defines empty space around the imaging data.

scale

Numeric scaling factor for pixel dimensions (default: 1). Values >1 enlarge, <1 shrink while maintaining original proportions.

is.multiple.combine.wide

Logical indicating if wide-format multi-sample layout should be prioritized (default: FALSE). When FALSE, attempts to maintain 1:1 aspect ratio by applying ratio scaling when needed.

ratio.threshold

Numeric threshold for aspect ratio adjustment (default: 1.25). Triggers dimension scaling when width/height ratio exceeds this value.

ratio.scale

Multiplicative factor (default: 1.5) applied to under-dimensioned axis when aspect ratio exceeds threshold.

Details

Authors

xieguigang

Value

A list containing: \itemize{ \item size - Numeric vector [width, height] of recommended canvas dimensions \item scale - Applied scaling factors (if any) }

Examples

# Basic usage autoSize(c(1024, 768), padding = c(20, 15, 20, 15))
